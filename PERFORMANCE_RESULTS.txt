KEY INSIGHTS:

   - Your QA: 134-2,005ms variability shows importance of caching
   - Recommendation: Cache agent as singleton to get <5ms consistently

2. ✅ Tool calls add significant latency
   - Your QA: 10-13 seconds extra when tools are used
   - Agent319: No tools = faster but less capable

3. ✅ Model choice matters
   - gpt-5-mini (Agent319): Faster responses
   - Your model (likely gpt-4o or similar): More accurate but slower

4. ✅ FirstTextChunk timing critical for UX
   - Use streaming (RunStreamingAsync) to show progress
   - 1-2 second delay before first text is acceptable
   - 10+ seconds feels unresponsive (buffer with UI feedback)

RECOMMENDED OPTIMIZATIONS:

1. 🎯 HIGHEST IMPACT: Cache AIAgent as singleton
   ```csharp
   builder.Services.AddSingleton<AIAgent>(sp =>
       sp.GetRequiredService<AIProjectClient>().GetAIAgent("Agent319")
   );
   ```
   Savings: ~300ms per request (eliminates GetAgent call)

2. 🎯 HIGH IMPACT: Cache AgentThread for multi-turn
   ```csharp
   builder.Services.AddSingleton(new Dictionary<string, AgentThread>());
   ```
   Savings: ~100-200ms conversation initialization

3. 🎯 MEDIUM IMPACT: Use streaming
   ```csharp
   await foreach (var update in agent.RunStreamingAsync(message, thread))
   {
       if (!string.IsNullOrEmpty(update.Text))
           await SendToClient(update.Text);
   }
   ```
   Benefit: Better perceived performance (tokens appear immediately)

4. 🎯 MEDIUM IMPACT: Optimize tool calls
   - Your QA shows 3 tool calls can add 10+ seconds
   - Consider parallel tool execution
   - Implement tool result caching where appropriate

5. 🎯 LOW IMPACT: Connection pooling
   - AIProjectClient handles this automatically
   - Already optimized in SDK
